UTF-8 인코딩을 위해 무엇이 바뀌었는지 설명합니다.

파일 설명

  • shiftjis.xml, kr enums.cpp 두 파일은 이전 버전과의 호환성을 위해 ANSI 인코딩을 유지합니다.
  • ini 파일은 ANSI 버전과의 호환성을 위해 UTF-8 BOM 이 반드시 필요합니다.
    xml, cpp 파일은 BOM 이 꼭 필요하지 않지만 사용을 권장합니다.
  • xml 파일에서 UTF-8 인코딩을 사용할 경우 루트 노드에 utf8 속성 값을 설정해야합니다.
    UTF-8로 인코딩된 cpp 파일에서 pk::load_xml 함수를 사용할 때도 마찬가지입니다.
  • ANSI 인코딩을 사용하는 파일은 콘솔창에서 제대로 표시되지 않습니다.

문자열

  • pk::draw_text, pk::message_box, pk::balloon 등 게임 안에서 문자열을 표시하는 함수의 경우 문자열의 시작이 '\xff\x??'일 경우 해당 언어로 표시할 수 있습니다.
    다만 encoding.xml에서 설정한 폰트 파일이 필요합니다.
    pk::draw_text("\xff\x01" + pk::u8encode("テスト", LANG_JP), pk::point(100, 200), -1, FONT_BIG, 0xffff0000);
    pk::draw_text("\xff\x02" + pk::u8encode("테스트", LANG_KR), pk::point(100, 250), -1, FONT_BIG, 0xffff0000);
    pk::draw_text("\xff\x03" + pk::u8encode("测试", LANG_SC), pk::point(100, 300), -1, FONT_BIG, 0xffff0000);
